# Security Release Gate — Regression Checklist

> **Purpose**: Verify these items before merging to `main` / cutting a release.  
> Every item must be ✅ (pass) or explicitly risk-accepted with a comment.

---

## 1. Automated CI gates (must be green)

| Check | Command / Workflow |
|-------|--------------------|
| CSRF same-origin guard | `npm run check:csrf` · GH Actions `csrf-guard` job |
| TypeScript type-check | `npx astro check` · GH Actions `typecheck` job |
| Build passes | `npm run build` |

---

## 2. Authentication & session

- [ ] Cookie-authenticated POST/PUT/PATCH/DELETE routes call `ensureSameOrigin(request)` as the **first** statement.
- [ ] `sb-access-token` and `sb-refresh-token` cookies are `HttpOnly`, `Secure` (forced `true` in production), `SameSite=Lax`.
- [ ] Session termination scopes the DB update by **both** `sessionId` and `userId` (prevents IDOR).
- [ ] Admin login rate-limit / lockout logic is intact after changes to `auth/` routes.
- [ ] MFA enroll / verify / unenroll endpoints still call `ensureSameOrigin`.

---

## 3. Authorization

- [ ] No admin route returns data or performs writes before verifying an active admin session.
- [ ] Row-Level Security (RLS) policies in Supabase are still applied after any migration; run `fix_rls_all_tables.sql` if tables were altered.
- [ ] Client-supplied `userId` / `orderId` is cross-checked against the authenticated session before use.

---

## 4. Input validation & injection

- [ ] All user-supplied fields are validated at the API boundary before insertion into Supabase.  
  No raw string interpolation into SQL or dynamic query builders.
- [ ] File uploads enforce allowlisted MIME types, maximum file size, and safe destination paths (no path traversal via `../`).
- [ ] Webhook endpoints (payment providers, push notifications) verify the request signature before processing.

---

## 5. Secrets & configuration

- [ ] No secrets, tokens, or credentials are present in committed source files.  
  Run: `git grep -rn "sk_live\|sk_test\|PRIVATE_KEY\|password\s*=\s*['\"]" -- '*.ts' '*.mjs' '*.cjs'`
- [ ] All sensitive values are read exclusively from `import.meta.env` / `process.env`; `.env` is in `.gitignore`.
- [ ] VAPID private key is stored only in an environment variable, never logged.

---

## 6. Error handling & information leakage

- [ ] API error responses return generic messages to clients; detailed errors go to server-side logs only.
- [ ] No stack traces, internal table names, or Supabase error objects are forwarded to the browser.
- [ ] `logger.ts` redacts tokens, raw card data, and full PII before writing to the activity log.

---

## 7. Security headers & CORS

- [ ] `Content-Security-Policy`, `X-Content-Type-Options`, and `Referrer-Policy` headers remain configured.
- [ ] `Access-Control-Allow-Origin: *` is **not** set on cookied / authenticated endpoints.
- [ ] `astro.config.mjs` has no overly permissive CORS or CSP relaxation introduced since the last release.

---

## 8. Dependencies

- [ ] `npm audit --audit-level=high` returns zero high/critical vulnerabilities.
- [ ] No newly added package is unpinned or pulls in a known-vulnerable transitive dependency.
