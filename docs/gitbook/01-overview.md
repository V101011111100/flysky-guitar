# Tổng quan phạm vi tài liệu

## Mục tiêu
- Xác định rõ phạm vi tài liệu vận hành cho Admin và Nhân viên.
- Chuẩn hóa quy trình thao tác theo hướng ngắn gọn, dễ thực hiện.
- Đồng bộ cách vận hành giữa frontend, backoffice và các API phục vụ nghiệp vụ.

## Phạm vi hệ thống

### Frontend khách hàng
- Trang chủ, danh mục, chi tiết sản phẩm, giỏ hàng, thanh toán, trang xác nhận đơn.
- Trang liên hệ, blog công khai, các widget hỗ trợ nhanh.

### Admin backoffice
- Dashboard, sản phẩm, tồn kho, đơn hàng, vận chuyển.
- Reviews, consultations, media, blog CMS.
- Marketing, discount, SEO, newsletter.
- POS, phân quyền, activity log, nhóm cài đặt hệ thống.

### API phục vụ vận hành
- Contact, newsletter, orders, products, media.
- Settings, payment settings, shipping, push notifications.
- Auth/session/MFA, reviews, blog, templates.

## Nguyên tắc tài liệu
- Bám sát chức năng đang có trong hệ thống.
- Ưu tiên SOP và checklist hơn mô tả lý thuyết.
- Mỗi chương cần có phần lỗi thường gặp và cách xử lý.
